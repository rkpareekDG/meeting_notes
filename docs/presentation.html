📹 Meeting Processing Flow

1
Meeting Ends
User hosts Zoom meeting with recording enabled
↓
2
Webhook Received
Zoom sends recording.completed event to bot
↓
3
Download Transcript
Bot downloads using user's OAuth token
4
AI Analysis
GPT-4 generates summary + action items
↓
5
Post to Slack
Rich message with summary + buttons
↓
6
User Actions
Create tickets, schedule follow-ups

🔒 Security Features

Feature Implementation Benefit
OAuth 2.0 User-based authorization flow No passwords stored, users control access
Token Encryption AES-256-GCM with unique nonces Tokens encrypted at rest
Webhook Verification HMAC signature validation Prevents fake webhook attacks
CSRF Protection State parameter with expiry Prevents authorization hijacking
Scoped Permissions Minimal OAuth scopes requested Least privilege principle

🚀 Quick Start

1. Clone & Install

git clone <repository> cd ai-meeting-bot-python python -m venv venv source venv/bin/activate pip install -e .

2. Configure

cp .env.example .env # Edit .env with your credentials

3. Start ngrok

ngrok http 3000 # Copy the https URL

4. Run Server

python run.py # Server at http://localhost:3000

5. Test in Slack

@MeetingBot setup
